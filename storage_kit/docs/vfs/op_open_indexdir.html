op_open_indexdir - Creates a handle for an indexdir.

Creates a handle for an indexdir, in the form of the returned cookie. This is then used for other calls on this indexdir.

Declaration

int op_open_indexdir(void *ns, void **cookie);

Parameters

Inputs

ns - pointer to a volume structure

Outputs

cookie - The cookie which can be used as a handle on the indexdir

Return

B_OK - Operation completed
ENOMEM - Not enough memory
EINVAL - Invaild parameters
ENOENT - The file to which the attribute is part of does not exist.
ENOSPC - No space on device.

Notes

This definition was devised using the source for the AtheOS file system driver (downloadable from bebits.com) and OpenBeFS from the OpenBeOS CVS.
