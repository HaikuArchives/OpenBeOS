op_read_query - Reads the next set of results from the query.

Reads the set of results in the query, and returns them as dirents. The number of dirent's is specified by num.

Declaration

int op_read_query(void *ns, void *cookie, long *num, struct dirent *buf, size_t bufsize);

Parameters

Inputs

ns - pointer to a volume structure
cookie - query favoured cookie
bufsize - unknown

Outputs

num - number of dirents returned
buf - query results

Return

B_OK - Operation completed
ENOMEM - Not enough memory
EINVAL - Invaild parameters
ENOENT - The file to which the attribute is part of does not exist.
ENOSPC - No space on device.

Notes

This definition was devised using the source for the AtheOS file system driver (downloadable from bebits.com) and OpenBeFS from the OpenBeOS CVS.
