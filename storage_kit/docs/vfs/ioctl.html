ioctl -- catch-all non-standard feature interface


Used to perform non-standard, filesystem specific operations on files

Declaration

int ioctl (     void  * pVolume,
  void  * pNode,
  void  * pCookie,
  int   nCommand,
  void  * pBuffer,
  size_t   nLen   );

Parameters

Input:
pVolume -- Pointer to the file system specific volume structure returned by mount
pNode -- Pointer to the node of the file
pCookie -- Pointer to the cookie for the file
nCommand -- The file system specific command that ioctl should perfom
pBuffer -- The address of a chunk of memory used for passing parameters to (and maybe from) ioctl
nLen -- Length of the chunk of memory pointed to by pBuffer

Return

B_OK -- The function completed successfully

Notes

This description derived from the sources for OpenBFS and AtheosFS, as well as parts of Practical Filesystem Design by Dominic Giampoalo
